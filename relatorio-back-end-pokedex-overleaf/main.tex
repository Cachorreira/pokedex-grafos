\section{Descrição da Aplicação}

A aplicação desenvolvida é uma Pokedex em Grafos. Uma Pokedex, no contexto do universo Pokémon, funciona como um catálogo digital no qual são registradas informações sobre diferentes Pokémon, como nome, número, imagem, descrição, tipo, habilidades e evolução. No sistema desenvolvido, esse conceito foi adaptado para uma aplicação web com front-end, back-end e banco de dados integrados.

O objetivo do sistema é permitir que o usuário visualize Pokémon, consulte suas relações e interaja como treinador. A aplicação permite login, cadastro de usuário, cadastro de Pokémon, edição, remoção, favoritos e criação de equipes. O diferencial do projeto está no uso do banco Neo4j, que armazena os dados como nós e relacionamentos. Assim, um Pokémon não aparece apenas como um registro isolado, mas como uma entidade conectada a tipos, fraquezas, habilidades, evoluções, usuários e equipes.

A arquitetura geral do sistema é composta por um front-end em HTML, CSS e JavaScript puro, uma API REST desenvolvida com Node.js e Express, autenticação com JWT, armazenamento de senhas com bcrypt e persistência de dados em Neo4j.

\section{Exemplos de Implementação}

\subsection{Conexão com o Neo4j}

\begin{lstlisting}[language=JavaScript]
const driver = neo4j.driver(
  process.env.NEO4J_URI,
  neo4j.auth.basic(process.env.NEO4J_USER, process.env.NEO4J_PASSWORD)
);
\end{lstlisting}

Esse trecho mostra que o back-end se conecta ao Neo4j usando variáveis de ambiente. Isso evita deixar configurações sensíveis fixas no código.

\subsection{Proteção de Rotas com JWT}

\begin{lstlisting}[language=JavaScript]
if (!authHeader || !authHeader.startsWith('Bearer ')) {
  return res.status(401).json({ erro: 'Token nao fornecido.' });
}

const token = authHeader.split(' ')[1];
const payload = jwt.verify(token, process.env.JWT_SECRET);
req.usuarioId = payload.id;
next();
\end{lstlisting}

O middleware verifica se a requisição possui token. Se o token for válido, o id do usuário fica disponível para os controllers.

\subsection{Uso de bcrypt}

\begin{lstlisting}[language=JavaScript]
const senhaHash = await bcrypt.hash(senha, 10);
const senhaCorreta = await bcrypt.compare(senha, usuario.senhaHash || '');
\end{lstlisting}

O bcrypt é usado para salvar apenas o hash da senha e para comparar a senha digitada no login com o hash armazenado.

\subsection{Criação de Relacionamento no Grafo}

\begin{lstlisting}[language=SQL]
MATCH (u:Usuario {id: $usuarioId}), (p:Pokemon {id: $pokemonId})
MERGE (u)-[:FAVORITOU]->(p)
RETURN p
\end{lstlisting}

Esse exemplo representa a operação de favoritar um Pokémon. No Neo4j, o favorito é uma relação entre usuário e Pokémon.

\subsection{Consumo da API pelo Front-End}

\begin{lstlisting}[language=JavaScript]
async function api(path, options = {}) {
  const headers = { 'Content-Type': 'application/json', ...(options.headers || {}) };
  if (state.token) headers.Authorization = `Bearer ${state.token}`;

  const response = await fetch(`${API_URL}${path}`, { ...options, headers });
  const payload = await response.json().catch(() => ({}));

  if (!response.ok) {
    const details = payload.erro || payload.erros?.join(' ') || 'Falha na requisicao.';
    throw new Error(details);
  }

  return payload;
}
\end{lstlisting}

Essa função centraliza o consumo da API. Ela envia o token quando existe, processa JSON e transforma erros da API em mensagens para a interface.

\section{Conclusão}

O sistema Pokedex em Grafos atende aos requisitos obrigatórios do trabalho. A aplicação possui mais de três recursos, CRUD completo para Pokémon e Tipo, relacionamento entre recursos, conexão com banco Neo4j, validação de entrada, middleware de log, tratamento centralizado de erros, autenticação com JWT, rotas protegidas e senhas armazenadas com bcrypt.

No front-end, a aplicação utiliza HTML, CSS e JavaScript puro, consome a API com \texttt{fetch}, armazena token JWT no \texttt{localStorage}, envia o token em requisições protegidas e exibe mensagens de erro retornadas pela API. A modelagem em grafo se mostrou adequada ao tema, pois permite representar de forma natural as relações entre Pokémon, tipos, habilidades, evoluções, favoritos e equipes.
